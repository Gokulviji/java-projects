KEY WORDS

VAR:Declares a variable

In JavaScript, the var keyword is used to declare variables. However, it's important to note that var has some quirks and behaviors that may lead to unexpected results, especially when compared to the newer variable declaration keywords introduced in ES6 (ECMAScript 2015), such as let and const.

1.Function Scope: Variables declared with var are function-scoped. This means that they are accessible anywhere within the function in which they are declared, regardless of block scopes (like if statements or loops).

2. Hoisting: Variables declared with var are hoisted to the top of their function or global scope. This means that the variable declaration is effectively moved to the top of its scope during the compilation phase, although its assignment (if any) remains in place. This can sometimes lead to unexpected behavior, especially for those unfamiliar with the concept.

3. No Block Scope: Unlike let and const, var does not have block-level scope. This means that variables declared with var inside blocks (like if statements or loops) are accessible outside of those blocks.

4.Re-declaration: Variables declared with var can be re-declared within the same scope without causing an error. This can sometimes lead to unintended consequences and bugs.

LET:Declares a Block variable

In JavaScript, the let keyword was introduced in ECMAScript 2015 (ES6) to provide block-scoped variable declarations. Unlike the var keyword, which has function-level scope, let allows variables to be scoped to the block in which they are defined, such as within an if statement or a loop. This behavior helps prevent certain bugs and unintended side effects that can occur with var.

1.Block Scope: Variables declared with let are scoped to the nearest enclosing block (i.e., any pair of {}). This means that they are only accessible within that block and any nested blocks.

2.No Hoisting: Variables declared with let are not hoisted to the top of their scope. They are only accessible after the declaration statement has been executed. This helps prevent the "temporal dead zone" that can occur with var.

3.No Re-declaration: Unlike var, you cannot re-declare a variable with the same name within the same block scope using let. Attempting to do so will result in a syntax error.

4.Initialization: Variables declared with let can be declared without being immediately initialized. They will be initialized with the value undefined until they are assigned a value.

5.Can be Mutable: Unlike const, variables declared with let can be reassigned a new value after declaration.

CONST:Declares a Block Constant

In JavaScript, the const keyword was introduced in ECMAScript 2015 (ES6) alongside let to declare constants. Constants are variables whose values cannot be reassigned after initialization. They are useful for declaring values that should remain constant throughout the execution of a program.

1.Constant Value: Variables declared with const must be initialized with a value, and once assigned, their value cannot be changed through reassignment. This means that attempting to assign a new value to a const variable after initialization will result in a syntax error.

2.Block Scope: Like let, variables declared with const are block-scoped. They are only accessible within the block in which they are defined, and any nested blocks.

3.No Hoisting: const variables are not hoisted to the top of their scope, similar to let. They must be declared before they are accessed in the code.

4.No Re-declaration: Just like let, you cannot re-declare a variable with the same name within the same block scope using const. Attempting to do so will result in a syntax error.

5.Immutable Binding: While the value of a const variable cannot be reassigned, it does not mean that the value itself is immutable. If a const variable holds an object or array, the properties or elements of that object or array can still be mutated.
